# Шаблон Word-документа

Структура запроса на разъяснение, который генерирует [generate_request.py](generate_request.py).

---

## Параметры документа

- **Шрифт:** Times New Roman, 12 pt (основной текст)
- **Размер страницы:** А4 (210 × 297 мм)
- **Поля:**
  - Сверху: 2 см
  - Снизу: 2 см
  - Слева: 3 см (для подшивки)
  - Справа: 1.5 см
- **Межстрочный интервал:** 1.15
- **Выравнивание основного текста:** по ширине

---

## Что в документе ЕСТЬ и чего нет

### Есть

```
[ЗАГОЛОВОК — по центру, жирный]
ЗАПРОС НА РАЗЪЯСНЕНИЕ
положений извещения о проведении закупки
№ [реестровый номер]

[ОБРАЩЕНИЕ — слева]
Уважаемые специалисты!

[ВСТУПИТЕЛЬНАЯ ФРАЗА]
Прошу разъяснить положения извещения о проведении закупки
по следующим вопросам.

[БЛОКИ ВОПРОСОВ — пронумерованы]
1. [Подзаголовок]
   [Текст вопроса 1]

2. [Подзаголовок]
   [Текст вопроса 2]

3. [Подзаголовок]
   [Текст вопроса 3]

[ПОДПИСЬ — слева]
С уважением,
[ФИО]

[ДАТА — слева, отдельной строкой]
ДД.ММ.ГГГГ
```

### Чего НЕТ

❌ **Шапка с адресатом и заявителем.** Заказчик и заявитель — оба прикрепляются автоматически при подаче через ЕИС. В файл их вписывать не нужно.

❌ **Реквизиты:** ИНН, КПП, юр. адрес, ОГРН, телефон, e-mail. Это всё подаёт площадка.

❌ **Логотипы**, угловой штамп, реквизиты банка — не нужны для делового запроса.

## Полный пример с тремя вопросами

```
              ЗАПРОС НА РАЗЪЯСНЕНИЕ
   положений извещения о проведении закупки
            № 0121600013926000030


Уважаемые специалисты!

Прошу разъяснить положения извещения о проведении закупки
по следующим вопросам.


1. Эквивалент по позиции 4 Локального сметного расчёта

В Локальном сметном расчёте (Приложение № 2 к извещению),
позиция № 4, указано оборудование «видеокамера TRASSIR
TR-D2181IR3v3 производства компании DSSL».

Согласно п. 4.1 Технического задания (Приложение № 1
к извещению) допускается поставка эквивалентного оборудования.

Прошу подтвердить, что нашей компании будет принято на приёмку
оборудование иного производителя — модель Foton EI8012CP1-D —
при условии, что все указанные в Техническом задании
характеристики выполняются не хуже, чем у указанной модели.


2. Коллизия по объёму архивного хранилища

В Техническом задании (Приложение № 1, п. 5.3) указано:
«объём архива видеонаблюдения должен составлять не менее 32 ТБ
с обеспечением хранения записей в течение 30 суток».

В Локальном сметном расчёте (Приложение № 2, позиция № 14)
предусмотрена поставка двух жёстких дисков ёмкостью 8 ТБ
каждый — суммарно 16 ТБ.

Прошу уточнить, какой объём дискового пространства является
обязательным для приёмки:

а) 32 ТБ, как в Техническом задании;
б) 16 ТБ, как в Локальном сметном расчёте.


3. Объём прокладки кабельной линии

В Локальном сметном расчёте (позиция № 22) предусмотрена
прокладка кабельной линии длиной 850 метров. Согласно схеме
расположения камер на объекте, суммарная длина трасс
не превышает 600 метров.

Прошу уточнить, какой объём прокладки кабеля является
обязательным для приёмки:

а) 850 метров, как в смете;
б) фактический объём по итогам монтажа.


С уважением,

И.И. Иванов

02.05.2026
```

## Особенности оформления

### Заголовок

- Выравнивание — по центру.
- Шрифт — Times New Roman, 14 pt, **жирный** для слова «ЗАПРОС НА РАЗЪЯСНЕНИЕ» и для номера закупки.
- Подпись «положений извещения о проведении закупки» — обычным начертанием.

### Обращение и вступление

- Слева, обычным начертанием.
- Между ними — пустая строка.
- Между вступлением и первым блоком — пустая строка.

### Блоки вопросов

- Каждый блок начинается с **подзаголовка** (например, «1. Эквивалент по позиции 4 ЛСР») — жирным начертанием.
- Тело блока — обычным, выравнивание по ширине.
- Цитаты из документации — *курсив* в кавычках. Пример: *«объём архива не менее 32 ТБ»*.
- Между блоками — пустая строка для визуального разделения.

### Подпись

- «С уважением,» — отдельной строкой слева.
- ФИО подписанта — отдельной строкой слева, обычным начертанием.
- Должность не нужна (её прикрепит ЕИС).

### Дата

- Внизу слева, под подписью, отдельной строкой.
- Формат: ДД.ММ.ГГГГ (например, 02.05.2026).

## Имя файла

**Шаблон:** `Запрос разъяснений по закупке № [рег.№].docx`

**Пример:** `Запрос разъяснений по закупке № 0121600013926000030.docx`

Если вторая итерация запросов по той же закупке (после ответа на первый) — добавляется номер:

- `Запрос разъяснений № 2 по закупке № 0121600013926000030.docx`

---

## Что НЕ включаем в документ

- Шапку с реквизитами обеих сторон (площадка их прикрепит)
- Логотипы компании
- Коды ОКВЭД, ОГРН
- Ссылки на договоры с заказчиком
- Длинные правовые обоснования (это для жалобы, не для запроса)
- Копии каких-либо документов (если нужны — отдельным файлом-приложением)
