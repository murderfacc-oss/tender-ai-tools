# Пять сюжетов вопросов

Все вопросы по одной закупке упаковываются **в один документ** — каждый отдельным пронумерованным блоком. Это связано с лимитом запросов (до 3 от одного лица для электронного аукциона) — сам файл считается за один запрос.

Готовые блоки ниже формулируются так, чтобы их можно было ставить друг за другом внутри одного документа без повторов и внутренних противоречий.

---

## Структура общего документа

```
[Заголовок]

ЗАПРОС НА РАЗЪЯСНЕНИЕ
положений извещения о проведении закупки № [рег.№]


[Вступительная фраза]

Уважаемые специалисты!

Прошу разъяснить положения извещения о проведении закупки
по следующим вопросам.


[Блок 1]

1. [Краткий заголовок вопроса — необязательно, но удобно]

[Текст блока 1 по выбранному сюжету.]


[Блок 2]

2. [Краткий заголовок вопроса]

[Текст блока 2.]


[Блок 3]

3. ...


[Подпись]

С уважением,
[ФИО]

[Дата]
```

## 1. Эквивалент

### 1A. Подтверждение эквивалента (общая формулировка)

```
1. Эквивалент по позиции [N] Локального сметного расчёта

В Локальном сметном расчёте (Приложение № [N] к извещению),
позиция № [N], указано оборудование «[наименование, модель,
производитель — точно из документа]».

Согласно п. [N] Технического задания (Приложение № [N]
к извещению) допускается поставка эквивалентного
оборудования.

Прошу подтвердить, что нашей компании будет принято на
приёмку оборудование иного производителя при условии,
что все указанные в Техническом задании характеристики
выполняются не хуже, чем у указанной модели.
```

**Закрытый формат ответа:** «да» / «нет, нужны уточнения».

**Что мы получаем:**
- «Да, принимается» — это защитный документ при приёмке. Заказчик потом не может предъявить «не Hikvision».
- «Нет» с обоснованием — у нас есть основания для жалобы по ст. 33.
- Уход от ответа — материал для жалобы.

---

### 1B. Подтверждение конкретной модели

Применяется, когда мы уже знаем, какую конкретно модель планируем ставить, и хотим закрепить её совместимость.

```
[N]. Соответствие модели [производитель — модель] требованиям ТЗ

В Техническом задании пункт № [N] указано требование
к [видеокамере / регистратору / коммутатору]:
«[точная цитата из ТЗ]».

Прошу подтвердить, что оборудование [производитель — модель],
имеющее следующие характеристики:
- [характеристика 1: число с единицами]
- [характеристика 2: число с единицами]
- [характеристика 3]

соответствует указанному в Техническом задании требованию.
```

**Когда применять:** когда уверены в выборе модели и хотим закрыть тему до подачи заявки.

---

### 1C. Эквивалент в позиции, где «или эквивалент» НЕ указано

Применяется, когда в ТЗ или ЛСР указан конкретный бренд **без** формулы «или эквивалент». Это уже почти жалоба, но мягкая.

```
[N]. Возможность применения эквивалента по позиции № [N]

В позиции № [N] Локального сметного расчёта указан конкретный
товарный знак «[бренд]» без сопровождения словами «или
эквивалент».

Прошу уточнить:

а) допускается ли поставка оборудования иного производителя
   с не худшими характеристиками;

б) если эквивалент не допускается — на каком из перечисленных
   в подпунктах «б»–«д» пункта 1 части 1 статьи 33 Закона
   № 44-ФЗ оснований установлено такое ограничение
   (несовместимость с существующим оборудованием, ремонт
   существующей техники, медицинские показания).
```

**Что мы получаем:**
- Заказчик меняет извещение — добавляет «или эквивалент».
- Заказчик отвечает с обоснованием — проверяем, действительно ли применимо.
- Заказчик отказывается отвечать — есть документально зафиксированный материал для жалобы.

## 2. Коллизия

### 2A. Расхождение между ТЗ и ЛСР по объёму

Применяется при расхождении в количестве, ёмкости, длине, массе.

```
[N]. Коллизия между ТЗ и ЛСР по [параметру]

При изучении документации закупки выявлено расхождение между
Техническим заданием и Локальным сметным расчётом:

В Техническом задании (Приложение № [N]), пункт № [N], указано:
«[точная цитата с числами]».

В Локальном сметном расчёте (Приложение № [N]), позиция № [N],
указано: «[точная цитата с числами]».

Прошу уточнить, какое из требований является обязательным
для исполнения:

а) объём, указанный в Техническом задании;
б) объём, указанный в Локальном сметном расчёте.

Этот вопрос задан для корректного формирования заявки и расчёта
стоимости работ.
```

**Закрытый формат ответа:** «вариант а» / «вариант б».

---

### 2B. Расхождение между Описанием объекта закупки и проектом контракта

```
[N]. Коллизия между Описанием объекта закупки и проектом контракта

В Описании объекта закупки (раздел [N], стр. [N]) указано:
«[точная цитата]».

В проекте контракта (Приложение № [N]) пункт № [N] содержит
иное условие: «[точная цитата]».

Прошу уточнить, какое из условий является применимым:

а) указанное в Описании объекта закупки;
б) указанное в проекте контракта.
```

---

### 2C. Расхождение между ОКПД2 и фактическим описанием работ

Применяется, когда код ОКПД (например, 33.20.42 — установка профессионального оборудования) не совпадает с тем, как описаны работы (как «капитальный ремонт» с СНиП и КС-2).

```
[N]. Применимость «строительных» требований при коде ОКПД2 [N.N.N]

Извещение о проведении закупки содержит код ОКПД2 [N.N.N]
«[наименование]». При этом проект контракта в пункте № [N]
требует от подрядчика [например: «оформление общего журнала
работ КС-6, актов скрытых работ, исполнительной документации
по СНиП»].

Прошу уточнить:

а) считаются ли указанные требования обязательными в полном
   объёме, как при выполнении строительных работ
   (глава 6 Градостроительного кодекса РФ); либо

б) указанные требования применяются формально, поскольку
   работы по коду ОКПД [N.N.N] не относятся к строительным.
```

**Что мы получаем:** заказчик либо снимает «строительные» требования (упрощает исполнение), либо подтверждает их (мы знаем, что готовить КС-6 и журналы).

## 3. Неоднозначность

### 3A. «Высокого качества», «современное», «надёжное»

```
[N]. Критерии проверки соответствия требованию «[требование]»

В пункте № [N] Технического задания указано требование:
«[точная цитата с расплывчатой формулировкой]».

Прошу уточнить, по каким конкретным показателям будет
проводиться проверка соответствия [товара / работ] указанному
требованию на этапе приёмки:

а) по конкретному перечню числовых характеристик
   (если да — прошу указать перечень);

б) по соответствию действующим государственным стандартам
   (если да — прошу указать ГОСТ).
```

---

### 3B. «Не хуже» без указания базы для сравнения

```
[N]. Минимально допустимое значение [параметра]

В пункте № [N] Технического задания указано требование
«не хуже [параметра]». При этом конкретное значение параметра,
относительно которого проводится сравнение, в документации
не указано.

Прошу уточнить, какое конкретное значение [параметра] является
минимально допустимым для приёмки:

а) [значение 1]
б) [значение 2]
```

---

### 3C. Требование без указания нормативного документа

Применяется, когда заказчик пишет «согласно требованиям пожарной безопасности» без ссылки на конкретный СП, или «согласно государственному стандарту» без указания номера ГОСТ.

```
[N]. Применимый нормативный документ к требованию «[требование]»

В пункте № [N] Технического задания указано требование
«[точная цитата]» без указания конкретного нормативного
документа.

Прошу уточнить, какой именно документ применяется при проверке
соответствия:

а) ГОСТ № [например, 51558-2014 — для систем видеонаблюдения];
б) СП [например, 484.1311500.2020 — для систем пожарной
   сигнализации];
в) иной документ — прошу указать.
```

## 4. Объём

### 4A. Расхождение количества единиц оборудования

```
[N]. Количество единиц [оборудования] для установки

Согласно Техническому заданию (Приложение № [N]) на объекте
монтажа предусмотрено [N] точек установки [например: камер
видеонаблюдения]. Согласно Локальному сметному расчёту
(Приложение № [N]), позиция № [N], указано количество [N]
штук того же оборудования.

Прошу уточнить, какое количество [камер] подлежит фактической
установке на объекте:

а) [число из ТЗ] штук;
б) [число из ЛСР] штук.
```

---

### 4B. Расхождение длины кабельных линий

```
[N]. Объём кабельной линии для приёмки

В Локальном сметном расчёте (позиция № [N]) предусмотрена
прокладка кабельной линии общей длиной [число] метров. Согласно
схеме расположения оборудования (Приложение № [N]), суммарная
длина трасс по объекту составляет около [число] метров.

Прошу уточнить, какой объём прокладки кабельной линии является
обязательным для приёмки:

а) [число из сметы] метров;
б) фактический объём, определяемый по итогам монтажа.
```

**Зачем спрашиваем:** если ответ «фактический объём», у нас есть основание подписать акт с фактическим, а не сметным метражом — без споров на приёмке.

## 5. Обоснование

### 5A. Снятое с производства оборудование

```
[N]. Замена снятого с производства оборудования по позиции № [N]

В Локальном сметном расчёте (позиция № [N]) указано
оборудование «[производитель] [модель]». Согласно открытой
информации производителя ([ссылка на сайт или цитата]), данная
модель снята с производства с [дата].

Прошу подтвердить:

а) допускается ли при исполнении контракта поставка иной
   актуальной модели того же производителя; либо

б) поставка модели иного производителя с не худшими
   характеристиками.
```

---

### 5B. Требование, по которому есть только один производитель

```
[N]. Возможность применения иных производителей по требованиям п. [N] ТЗ

В пункте № [N] Технического задания установлено требование:
«[точная цитата с конкретным сочетанием параметров]».

Согласно открытым данным каталогов производителей, указанному
сочетанию характеристик соответствует продукция единственного
производителя — [бренд].

Прошу уточнить:

а) какие иные производители, по сведениям заказчика, выпускают
   оборудование, соответствующее указанному сочетанию
   характеристик;

б) если такие производители отсутствуют — каким образом
   заказчиком учтены требования части 1 статьи 33 Закона
   № 44-ФЗ при формировании описания объекта закупки.
```

**Внимание:** этот блок — почти жалоба. Применять, только если уже решили подавать жалобу. Иначе можем «разбудить» заказчика, который перепишет ТЗ так, что станет хуже.

---

### 5C. Чрезмерные требования к опыту участника

```
[N]. Основание установления требования к опыту участника

В пункте № [N] извещения установлено требование к опыту
участника закупки: «[точная цитата]».

Прошу уточнить, на основании какого нормативного документа
установлено указанное требование:

а) Постановление Правительства РФ от 28.11.2013 № 2571
   (дополнительные требования);

б) часть 2 либо 2.1 статьи 31 Закона № 44-ФЗ;

в) иное основание — прошу указать.
```

## Полный пример документа с тремя вопросами

Это образец того, как несколько блоков объединяются в один файл (заголовок и подпись добавляет [generate_request.py](generate_request.py)):

```
              ЗАПРОС НА РАЗЪЯСНЕНИЕ
   положений извещения о проведении закупки
            № 0121600013926000030


Уважаемые специалисты!

Прошу разъяснить положения извещения о проведении закупки
по следующим вопросам.


1. Эквивалент по позиции 4 Локального сметного расчёта

В Локальном сметном расчёте (Приложение № 2 к извещению),
позиция № 4, указано оборудование «видеокамера TRASSIR
TR-D2181IR3v3 производства компании DSSL».

Согласно п. 4.1 Технического задания (Приложение № 1
к извещению) допускается поставка эквивалентного оборудования.

Прошу подтвердить, что нашей компании будет принято на приёмку
оборудование иного производителя — модель Foton EI8012CP1-D —
при условии, что все указанные в Техническом задании
характеристики выполняются не хуже, чем у указанной модели.


2. Коллизия по объёму архивного хранилища

В Техническом задании (Приложение № 1, п. 5.3) указано:
«объём архива видеонаблюдения должен составлять не менее 32 ТБ
с обеспечением хранения записей в течение 30 суток».

В Локальном сметном расчёте (Приложение № 2, позиция № 14)
предусмотрена поставка двух жёстких дисков ёмкостью 8 ТБ
каждый — суммарно 16 ТБ.

Прошу уточнить, какой объём дискового пространства является
обязательным для приёмки:

а) 32 ТБ, как в Техническом задании;
б) 16 ТБ, как в Локальном сметном расчёте.


3. Объём прокладки кабельной линии

В Локальном сметном расчёте (позиция № 22) предусмотрена
прокладка кабельной линии длиной 850 метров. Согласно схеме
расположения камер на объекте, суммарная длина трасс
не превышает 600 метров.

Прошу уточнить, какой объём прокладки кабеля является
обязательным для приёмки:

а) 850 метров, как в смете;
б) фактический объём по итогам монтажа.


С уважением,

И.И. Иванов

02.05.2026
```

## Общие правила тона

✅ «Прошу подтвердить»
✅ «Прошу уточнить»
✅ «Этот вопрос задан для…» (объяснить причину — это снижает конфликтность)
✅ «С уважением»

❌ «Требуем»
❌ «Считаем недопустимым»
❌ «Грубое нарушение»
❌ «Очевидно, что…»
❌ Длинные периоды с «между тем», «при этом», «с учётом изложенного»

---

## Что обязательно подкрепить документально

К каждому блоку:

1. **Точная цитата** из документа с указанием:
   - Названия документа («Техническое задание», «Локальный сметный расчёт»)
   - Номера приложения к извещению
   - Номера пункта или позиции
   - Страницы, если документ длинный

Проверка: если в блоке нет ссылок «согласно п. N документа Y» — переписать.
