## ✅ Сделано в v0.3.0 (2026-05-06)

### MCP `zakupki-eis` — печатная форма извещения

`download_tender` теперь дополнительно качает печатную форму со страницы
ЕИС (`/printForm/view.html?regNumber=...`) и сохраняет в корень папки:
- `print_form.html` — оригинал
- `print_form.txt` — чистый текст для LLM (без HTML-разметки)

Это сводная страница с заказчиком, объектом закупки, требованиями,
графиком — часто полнее, чем разрозненные приложения.

Реализация:
- `_get_print_form(reg_number, notice_type)` в `zakupki_scraper.py` —
  пробует URL для известного notice_type, затем универсальный, затем
  перебор по всем типам.
- `_save_print_form(html, folder)` — сохраняет HTML и извлекает чистый
  текст через BeautifulSoup (удаляет script/style/nav/header/footer).

Информация о печатной форме попадает в `.tender_meta.json` (поле
`print_form`), в вывод MCP-инструмента и читается скиллом `scan-zakupki`.

## ✅ Решено в 2026-05-11

### Вернули плагин как основной способ дистрибуции — v0.5.0

Прошлое решение «отказаться от плагина» было основано на ложной
предпосылке. Источник заблуждения — представление, что плагины «сложные»
из-за Python-зависимостей. На деле:

1. Claude Desktop с весны 2026 принимает плагины **через UI**: `Customize → Personal plugins → Create plugin → Upload plugin / Add marketplace`. Партнёр устанавливает за 30 секунд.
2. `launcher.py` уже решает проблему с pip-зависимостями MCP — автоустановка при первом запуске.
3. `${CLAUDE_PLUGIN_ROOT}` в `.mcp.json` делает путь портабельным — работает у любого пользователя без правок.

Что сделали:
- Перенесли `project/skills/` → `skills/` в корень для idiomatic plugin
  структуры.
- Создали `.claude-plugin/plugin.json` (манифест v0.5.0).
- Создали `.claude-plugin/marketplace.json` (каталог для Add marketplace
  через GitHub-URL).
- Создали `.mcp.json` в корне с `${CLAUDE_PLUGIN_ROOT}/mcp/launcher.py`.
- Переписали `scripts/build_plugin.py` — теперь это простой zip-builder
  по структуре репо, версия читается из `plugin.json` (один источник правды).
- `scripts/pack_skills.py` остался как **вспомогательный** для загрузки
  отдельных скиллов в Cowork.

Структура репо теперь = структуре плагина, никаких build-каталогов с
переименованием. См. README.md и INSTALL.md.
