## ✅ Сделано в v0.3.0 (2026-05-06)

### MCP `zakupki-eis` — печатная форма извещения

`download_tender` теперь дополнительно качает печатную форму со страницы
ЕИС (`/printForm/view.html?regNumber=...`) и сохраняет в корень папки:
- `print_form.html` — оригинал
- `print_form.txt` — чистый текст для LLM (без HTML-разметки)

Это сводная страница с заказчиком, объектом закупки, требованиями,
графиком — часто полнее, чем разрозненные приложения.

Реализация:
- `_get_print_form(reg_number, notice_type)` в `zakupki_scraper.py` —
  пробует URL для известного notice_type, затем универсальный, затем
  перебор по всем типам.
- `_save_print_form(html, folder)` — сохраняет HTML и извлекает чистый
  текст через BeautifulSoup (удаляет script/style/nav/header/footer).

Информация о печатной форме попадает в `.tender_meta.json` (поле
`print_form`), в вывод MCP-инструмента и читается скиллом `scan-zakupki`.

## ✅ Сделано в v0.4.0 (2026-05-06)

### scan-zakupki v0.5 — нативное чтение документов

`SKILL.md` обновлён: чтение `.pdf`/`.docx`/`.xlsx` идёт через pandoc и
нативное мультимодальное чтение модели вместо тяжёлой связки
`pdfplumber`/`easyocr`/`python-docx`. Сэкономили ~500 МБ
моделей easyocr и подняли качество (OCR теперь делает сама модель,
видя страницы как изображения).

Python-парсинг оставлен только как fallback для случаев, когда
pandoc/soffice недоступны.

Также: добавлена явная инструкция читать `print_form.txt` одним из
первых документов — это сводная карточка извещения, появляется
автоматически после `download_tender` v0.3+.

### MCP — чистка зависимостей

Из `launcher.py` REQUIRED убрана `python-docx` — MCP-сервер только
скрапит сайт, с Word-файлами не работает. Меньше зависимостей при
первом запуске.
