# SabyTrade MCP — контекст проекта

## Цель
Создать MCP-сервер который подключит SabyTrade к Claude Desktop.
Итоговая цель: закупка из Saby → автоматический анализ → решение «берёмся / не берёмся»

## Владелец проекта
Артём — монтаж слаботочных систем (видеонаблюдение, СКУД, речевое оповещение, СКС).
Участвует в закупках по 44-ФЗ и 223-ФЗ через площадку SabyTrade (saby.ru).
Не разработчик — объяснять просто, код писать с комментариями.

## Источник данных (изменение!)
Вместо SabyTrade используем DaMIA API-Закупки
- Документация: https://damia.ru/apizakupki
- Тариф: API-Старт (100 запросов на метод)
- Формат: REST/JSON
- Ключ: в файле .env (DAMIA_API_KEY)

## Стек
- Python (предпочтительно)
- MCP SDK: modelcontextprotocol (pip install mcp)
- DaMIA API: REST/JSON, аутентификация по API-ключу

## DaMIA API — основные методы (уточнить по документации)
- Поиск закупок по параметрам
- Получение деталей закупки по ID
- Аутентификация: API-ключ в заголовке каждого запроса

## MCP-инструменты которые нужно реализовать
- `search_tenders` — поиск закупок по ключевым словам / параметрам
- `get_tender_by_id` — получить детали конкретной закупки
- `get_tender_documents` — получить документацию закупки

## Важные ограничения
- Хранить DAMIA_API_KEY в .env, никогда не в коде
- Тариф API-Старт: 100 запросов на метод — экономить запросы
- Проверять методы по документации на damia.ru

## Как работать с этим проектом
Каждая сессия Claude Code = одна задача:
- Сессия «API» — тестируем запросы к DaMIA
- Сессия «Сервер» — пишем server.py
- Сессия «Подключение» — настраиваем Claude Desktop
- Сессия «Тесты» — проверяем всю связку
