# Отчёт 1 — Вердикт

## Назначение

Первый и обязательный отчёт по любой закупке. Даёт команде достаточно 
информации для решения «беремся / не беремся». AI описывает — человек решает.

Отчёт не содержит: подбора оборудования и эквивалентов, юридического 
анализа нарушений, рисков при исполнении, формулировок актов КС-2, 
оценок «выгодно / невыгодно», экономики, маржи, конкуренции, 
истории заказчика.

---

## Структура отчёта

### 1. Краткое резюме

Связный абзац 3–5 предложений. Только факты из документов. 
Никаких оценок от AI («интересная закупка», «стоит обратить внимание»).

Включает: кто заказчик, что закупается, НМЦК, способ закупки, 
ключевые особые требования если есть.

---

### 2. Карточка закупки

Оформляется таблицей.

#### Классификация

Описанием словами — не номером типа. Внутри скилл держит схему 4 типов:

| Внутренний тип | Признаки |
|---|---|
| Обычные работы/услуги | ОКПД2 33.xx, КВР 244/КОСГУ 226, конъюнктурный метод НМЦК, нет РД с чертежами |
| Квазистроительная | ОКПД2 43.xx, КВР 243/КОСГУ 225, но без полного пакета РД; смета на ремонтных расценках |
| Капремонт по строительной логике | ОКПД2 43.xx, КВР 243/КОСГУ 225, проектно-сметный метод НМЦК, есть РД и ВОР |
| Капремонт ОКС | То же + объект — капитальное строение, нужно СРО НОСТРОЙ |

В отчёт выводится: описание словами + обоснование по сигналам 
(ОКПД2, КВР/КОСГУ, метод НМЦК, наличие РД).

При пограничных случаях: описание ближайшего типа + строка «Оговорка:».

#### Портрет ожидаемого поведения заказчика

2–3 строки. Описывает: как заказчик воспринимает ЛСР (как обязательство 
по объёмам или только база НМЦК), будет ли требовать снизить цену при 
эквивалентах, объём строительной отчётности который потребует при сдаче.

#### Контакты заказчика

Название организации, ИНН (можно извлечь из ИКЗ — позиции 4–16), 
ФИО ответственного, телефон, email. Брать из извещения или печатной формы ЕИС.

#### Период исполнения

Как написано в документации + количество дней цифрой.  
Если в извещении и ТЗ сроки разные — показываем оба с пометкой источника.

#### География

Один адрес или список адресов объектов монтажа.

#### Тип закупки

Направление (видеонаблюдение / СКУД / СКС / комплекс) + краткое описание + 
цифры (из сметы при наличии, иначе из ТЗ/ЕИС).  
При расхождениях между сметой и ТЗ — пометка «см. Отчёт 3».  
При отсутствии расхождений — строка «расхождений между сметой и ТЗ не выявлено».

---

### 3. Особые требования

Список. Каждый пункт начинается с + (преференция/плюс для участника) 
или — (ограничение/барьер).

Проверяем и выводим:

- **СМП/СОНКО** — есть ли ограничение участия только для субъектов МСП
- **Национальный режим** — есть ли требование реестра Минпромторга 
  (постановления 878, 616 и др.); если есть — **+** со знаком плюс 
  (наша компания в реестре = преференция)
- **Лицензии** — ФСБ (если шифрование или гостайна), МЧС (если пожарка), 
  иные отраслевые
- **СРО** — требуется ли членство в НОСТРОЙ; на каком основании
- **Опыт** — требования по ч. 1.1 ст. 31 (если есть — сумма и тип контрактов)
- **Обеспечение заявки** — размер и форма
- **Обеспечение контракта** — размер и форма  
- **Срок подачи заявок** — если менее 7 суток от публикации до дедлайна, 
  выносить отдельной строкой с пометкой «⚠ сжатые сроки»

## Принцип квалификации закупки

Любая закупка на монтаж слаботочки до детального анализа квалифицируется 
по ключевому вопросу: **оформлена она по строительной логике или по логике 
обычных работ/услуг.**

От этого зависит:
- ЛСР как обязательство по объёмам или только база НМЦК
- Восприятие твёрдой цены при приёмке
- Право на экономию при эквиваленте без снижения цены
- Защитные документы при исполнении

Сигналы для квалификации:

| Сигнал | Строительная логика | Логика обычных работ |
|---|---|---|
| ОКПД2 | 43.xx | 33.xx |
| КВР/КОСГУ | 243/225 | 244/226 |
| Метод НМЦК | Проектно-сметный | Конъюнктурный |
| РД с чертежами | Есть | Нет |
| Расценки в смете | ГЭСНм, ФСНБ | Рыночные/коммерческие |

Результат квалификации — практическая установка для исполнения, 
не правовое заключение.

## Что НЕ входит в этот отчёт

- Подбор оборудования и эквивалентов → Отчёт 2
- Юридический анализ нарушений и противоречий → Отчёт 3
- Риски при исполнении, формулировки актов → Отчёт 4
- Жалобы, запросы разъяснений → Производный документ
- Оценки «выгодно / невыгодно», экономика, маржа → не делаем вообще
