# Чек-лист перед подачей жалобы

Финальная проверка. Скилл проходит её **после** написания черновика и **до** того, как сообщить пользователю «жалоба готова».

---

## Блок 1 — Право на подачу (с 2024 года)

С 2024 года ФАС не рассматривает жалобы от лиц, чьи права не могут быть нарушены действиями заказчика (ч. 1 ст. 105 44-ФЗ, разъяснение ФАС России от 16.02.2024). 10,6% жалоб в январе 2024 отклонены по этой причине.

**Проверить (для режима A):**

- [ ] Заявитель имеет лицензию / СРО / иное разрешение, требуемое в извещении? (Если требование лицензии есть, а у заявителя её нет — жалобу вернут.)
- [ ] Заявитель соответствует требованиям к опыту по ч. 1.1 ст. 31 (если установлены доптребования по ПП РФ от 28.11.2013 № 2571)?
- [ ] Заявитель — российское юрлицо / ИП (если установлен запрет иностранных)?
- [ ] У заявителя нет признаков, перечисленных в ст. 31 (банкротство, задолженность, иностранный оффшор)?

Если хотя бы один ответ «нет» — **предупредить пользователя**: «жалоба может быть возвращена без рассмотрения по причине отсутствия у заявителя правоспособности участвовать в закупке».

**Для режима C** этот блок не применяется (правоспособность — это сам факт быть стороной контракта).

**Для режима D** ограничение не действует (заявление по ст. 44 135-ФЗ может подать любое лицо).

---

## Блок 2 — Сроки

| Режим | Срок подачи | Отсчёт |
|---|---|---|
| A — документация | До окончания срока подачи заявок | Из извещения |
| A — действия комиссии при отборе | 5 раб. дней | С даты публикации протокола |
| B — отклонение заявки | 5 раб. дней | С даты публикации протокола |
| C — пояснения против РНП | До заседания комиссии РНП | Обычно 5–10 дней |
| D — антимонопольное заявление | Не привязан к торгам | — |

**Проверить:**

- [ ] Срок не пропущен.
- [ ] Указан запас минимум 1 рабочий день до дедлайна (на случай технических сбоев в ЕИС).

Если срок пропущен и нет уважительных причин — сообщить пользователю и предложить альтернативу (антимонопольное заявление по 135-ФЗ, иск в суд).

---

## Блок 3 — Обязательные реквизиты (ч. 8 ст. 105 44-ФЗ)

Без любого из них жалоба будет возвращена.

- [ ] Наименование контрольного органа (УФАС с указанием региона)
- [ ] Сведения о заявителе: наименование (ФИО для ИП), ИНН/КПП, адрес, контактный телефон
- [ ] ИКЗ (Идентификационный код закупки) — берётся из извещения
- [ ] Сведения о заказчике (наименование, ИНН) и об операторе ЭТП
- [ ] Перечень обжалуемых действий с конкретными доводами
- [ ] Реестровый номер контракта — **если контракт уже заключён**

**Дополнительно для режима C:**
- [ ] Указание на контракт: реестровый номер, дата, наименование заказчика
- [ ] Дата принятия решения об одностороннем отказе
- [ ] Дата вступления решения в силу

**Дополнительно для режима D:**
- [ ] Лица, в действиях которых усматриваются признаки нарушения (заказчик + участник)
- [ ] Конкретные нормы 135-ФЗ (ст. 16, 17 и/или 11)

---

## Блок 4 — Доказательная база

- [ ] Каждое обстоятельство в жалобе подкреплено документом.
- [ ] Все упомянутые документы перечислены в разделе «Приложения».
- [ ] Скриншоты из ЕИС содержат URL и дату.
- [ ] Цитаты в жалобе соответствуют тексту в документах (без переписывания «по смыслу»).
- [ ] Если есть запись разговоров — оформлены реквизиты (дата, время, длительность, идентификация сторон, расшифровка с таймкодами).
- [ ] Доверенность представителя приложена (если жалобу подписывает не директор / не сам ИП).

---

## Блок 5 — Структура и стиль

- [ ] Документ оформлен по [струтуре](struktura-zhaloby.md) (шапка → I → II → III → IV → V).
- [ ] Каждое обстоятельство — отдельный блок «факт → ↓ → вывод» в стиле [Усенкова](style-usenkova.md).
- [ ] Нет эмоциональных оценок («грубое», «явное», «безобразное»).
- [ ] Нет правовых лекций вне предмета (объяснений УФАС что такое 44-ФЗ).
- [ ] Нет неконкретных формулировок («как известно», «очевидно», «вероятно»).
- [ ] Каждая цитата — короткая, в кавычках, с источником.
- [ ] Просительная часть стандартная, без эмоций.

---

## Блок 6 — Канал подачи

- [ ] Подача только через ЕИС с УКЭП (бумажная не принимается с 2022).
- [ ] Контактные данные в жалобе позволяют УФАС связаться (e-mail работает, телефон доступен).
- [ ] Способ получения копии решения указан (ЕИС / e-mail / почта).

**Если у пользователя нет УКЭП ИП/ООО** — это блокирующая проблема. Сообщить и приостановить подачу до получения подписи.

---

## Блок 7 — Прогноз и предупреждения

Перед передачей жалобы пользователю — явно сообщить:

1. **Шанс обоснования** по аналогичным кейсам (см. таблицы в [argumenty-tipa-A.md](argumenty-tipa-A.md) и [argumenty-rnp.md](argumenty-rnp.md)).
2. **Возможные риски** для заявителя:
   - Отказ в рассмотрении (если есть проблемы по Блоку 1).
   - Парадокс «обоснована, но без предписания»: жалоба признаётся обоснованной, но если торги уже состоялись или нарушение не повлияло на результат — предписание не выдаётся. Сама обоснованность остаётся как защитный документ.
   - Возможная реакция заказчика (затягивание приёмки, более жёсткое поведение). Это особенно актуально, если заявитель — сторона контракта.
3. **Альтернативы**, если шанс низкий:
   - Запрос разъяснений вместо жалобы (для типа A на ранней стадии)
   - Иск в арбитражный суд (если речь о деньгах)
   - Антимонопольное заявление (если 44-ФЗ-жалоба не успевает)

---

## Финальное сообщение пользователю

Когда все блоки чек-листа пройдены, скилл выдаёт пользователю:

```
Черновик жалобы готов.

✅ Все обязательные реквизиты заполнены
✅ Срок не пропущен (до [дата] [время])
✅ Доказательная база собрана: [N] приложений
✅ Канал подачи: ЕИС с УКЭП

Прогноз по практике УФАС: [высокий / средний / низкий] шанс
обоснования. Опорные кейсы: [перечислить 1–2 близких из библиотеки].

Перед подачей рекомендую:
1. Прочитать черновик ещё раз с холодной головой
2. Проверить, нет ли в фактологии ошибок (даты, номера, цены)
3. Подписать УКЭП и подать через ЕИС не позднее [дата]

Если в ходе подачи возникнут технические вопросы — мы можем
добавить новые обстоятельства или уточнить просительную часть.
```

Если есть незакрытые пункты чек-листа — сообщить **точно какие** и предложить либо доработать, либо подать без них с предупреждением о рисках.
