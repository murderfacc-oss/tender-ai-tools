# zhaloba-fas — проект жалобы в УФАС

## Назначение

Готовит **проект текста** жалобы в территориальное УФАС (или антимонопольного заявления в ФАС) по делам, типичным для подрядчика на слаботочке: оспаривание документации до подачи заявок, защита от включения в РНП, заявление о сговоре заказчика и участника.

Скилл выдаёт готовый черновик в Markdown — пользователь переносит его в Word и подписывает УКЭП для подачи через ЕИС.

**Не делает:**
- Не подаёт жалобу автоматически (только пользователь, через ЕИС с подписью)
- Не выдаёт правовое заключение «вы выиграете / проиграете» — оценивает шансы по практике, но решение за пользователем
- Не заменяет юриста при сложных делах в арбитражном суде

---

## Три режима работы

### Режим A — жалоба на документацию закупки

Подаётся **до окончания срока подачи заявок** в УФАС по месту нахождения заказчика.

**Когда применять:**
- Заказчик использует товарный знак без слов «или эквивалент» (ст. 33)
- Совокупность характеристик соответствует одному производителю (ст. 33, ограничение конкуренции)
- Один код КТРУ для всех позиций включая разнотипные (ст. 7)
- Описание не по КТРУ при наличии кода (ст. 33 + Ростовское УФАС № 061/10/99-1516/2025)
- Нацрежим неправильно применён или не применён (ст. 14, ПП 1875)
- Право на односторонний отказ только в проекте контракта, не в извещении (ст. 42)
- Один ИНН в коммерческих предложениях обоснования НМЦК (ст. 22)
- Сжатые сроки подачи без объективных причин (ст. 42, 49)

**Источник аргументов:** [argumenty-tipa-A.md](references/argumenty-tipa-A.md)

---

### Режим C — пояснения в УФАС против РНП

Подаётся в УФАС по месту нахождения заказчика **до заседания комиссии РНП** (обычно 5–10 дней с момента поступления материалов от заказчика).

**Когда применять:**
- Заказчик принял одностороннее решение об отказе от исполнения контракта
- Материалы переданы в УФАС для рассмотрения вопроса о включении подрядчика в РНП
- Подрядчик считает, что включение неправомерно

**Стержневой тезис защиты:** РНП — исключительная мера, требует именно недобросовестности (умысел, уклонение, скрытие), а не любого нарушения срока или условия.

**Источник аргументов:** [argumenty-rnp.md](references/argumenty-rnp.md)

---

### Режим D — антимонопольное заявление по ст. 16, 17 135-ФЗ

Подаётся в УФАС как заявление о нарушении антимонопольного законодательства (ст. 44 135-ФЗ — это другая процедура, не ч. 1 ст. 105 44-ФЗ).

**Когда применять:**
- Все КП в обосновании НМЦК — от одного ИНН (физлицо/ИП)
- Тот же ИНН встречается среди участников аукциона
- В протоколе подачи цен — разрыв между «знакомыми» и аутсайдерами в десятки раз (схема «таран»)
- Зафиксированные в записи разговоров слова заказчика о знакомстве/личном интересе

**Параллельная стратегия:** одновременная подача в прокуратуру района по факту обоснования НМЦК (ст. 22 44-ФЗ) — усиливает позицию.

**Источник аргументов:** [sgovor-135fz.md](references/sgovor-135fz.md)

## Workflow

1. **Сбор входных данных** — пользователь предоставляет:
   - Документы закупки (извещение, ТЗ, ЛСР, проект контракта) — для режима A
   - Решение об одностороннем отказе, переписку, акты, экспертные заключения — для режима C
   - Протокол подачи цен, обоснование НМЦК, записи разговоров (если есть) — для режима D

2. **Уточняющие вопросы** — задать пользователю **до** написания черновика:
   - Кто заявитель: ИП/ООО, ИНН, КПП, адрес, контакт
   - Реестровый номер закупки, ИКЗ
   - Реквизиты заказчика и ЭТП
   - Реестровый номер контракта (если уже заключён)
   - Конкретные доводы пользователя (не просто «несправедливо», а **что именно** нарушено)
   - Какие документы есть на руках для приложения

3. **Проверка применимости** (особенно для типа A):
   - Имеет ли заявитель право подавать жалобу (с 2024 ФАС не рассматривает от лиц без правоспособности — см. [chek-list.md](references/chek-list.md))
   - Не пропущен ли срок (ч. 4 ст. 105 — до окончания подачи заявок для типа A; 2 рабочих дня после вступления решения о расторжении в силу для типа C)

4. **Подготовка тела жалобы** по [struktura-zhaloby.md](references/struktura-zhaloby.md):
   - Шапка (реквизиты сторон)
   - I. Сведения о закупке
   - II. Обстоятельства (формат «обстоятельство → ↓ → вывод»)
   - III. Правовая квалификация
   - IV. Просительная часть
   - V. Приложения

5. **Применение стиля Усенкова** — см. [style-usenkova.md](references/style-usenkova.md). Без эмоций, без правовых лекций вне предмета, цитаты с точными ссылками.

6. **Финальный чек-лист** — пройти [chek-list.md](references/chek-list.md) и явно сообщить пользователю:
   - Все ли обязательные реквизиты заполнены
   - Все ли приложения собраны
   - Срок подачи и канал подачи (ЕИС с УКЭП)
   - Прогноз исхода по практике УФАС (без гарантий)

## Принципы написания

- **Каждое обстоятельство — самостоятельная единица.** Один факт, одна ссылка на документ/пункт/дату, один вывод.
- **Все факты с привязкой к документу:** «Согласно п. 5.2 Приложения 3 к извещению…», «В соответствии с протоколом № … от …».
- **Цитаты — короткие**, в кавычках, с указанием источника.
- **Тон — профессиональный**, без оценочных слов «безобразие», «ужас», «грубое нарушение». Используем: «нарушает», «противоречит», «требует самостоятельной правовой оценки уполномоченным органом».
- **Без правовых лекций** вне предмета. Не объяснять УФАС, что такое 44-ФЗ.
- **«Совокупность» — ключевое слово** для режима A: «совокупность установленных характеристик соответствует исключительно товару одного производителя» — формула, через которую УФАС признаёт жалобу обоснованной по п. 1 ч. 1 ст. 33.

## Что НЕ входит в v0.1

- **Тип B** (жалоба на отклонение заявки или результаты торгов) — добавим, когда появится первый кейс в нашей практике
- **Автоматическая выгрузка в Word** с форматированием Усенкова (TNR 12, серые карточки, красные цитаты) — пользователь сейчас оформляет вручную; в v0.2 через python-docx
- **Подача в прокуратуру** как отдельная подзадача — упоминается в режиме D, но полноценный шаблон обращения в прокуратуру войдёт позже
- **Обжалование решения УФАС** в арбитражном суде — это уже работа юриста, не скилла
- **Иски по ст. 1102 ГК** (неосновательное обогащение) — это область скилла `yuridika`, не `zhaloba-fas`
